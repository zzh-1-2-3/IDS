## 系统概述

IDS-Web是一个基于深度学习的网络入侵检测系统的Web界面，集成了实时流量捕获、深度学习模型检测、威胁评估和自适应响应策略等功能。

### 核心功能
- **实时流量捕获**：基于Scapy抓取网络数据包
- **深度学习检测**：使用CNN模型进行流量分类
- **威胁评估**：多维度威胁评分算法
- **自适应响应**：根据攻击类型自动匹配并执行响应策略
- **可视化展示**：实时仪表盘、流量趋势、攻击分布图表
- **模型管理**：在线训练、模型切换、性能评估

## 环境要求

### 硬件要求
- **CPU**：Intel i5 或同等性能以上
- **内存**：8GB RAM（推荐16GB）
- **存储**：至少20GB可用空间
- **网络**：支持混杂模式的网卡（用于实时捕获）

### 软件要求

#### 操作系统
- Windows 10/11 或
- Ubuntu 20.04+ / CentOS 7+

#### Python环境
- Python 3.8+
- pip 包管理器

#### 数据库
- MySQL 5.7+ 或 MariaDB 10.3+

#### 其他依赖
- Node.js 14+（用于Chart.js）

## 部署步骤

### 1. 准备项目文件

将项目文件复制到本地目录：
```
C:\Users\lenovo\Desktop\03-基于深度学习的入侵检测系统网络流量分析系统IDS\IDS\IDS-Web
```

### 2. 安装Python依赖

#### 进入后端目录
```bash
cd IDS-Web/backend
```

#### 创建虚拟环境（推荐）
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows
venv\Scripts\activate

# Linux/macOS
source venv/bin/activate
```

#### 安装依赖包
```bash
pip install -r requirements.txt
```

### 3. 安装Chart.js

```bash
cd IDS-Web/chart
npm install
```

### 4. 配置MySQL数据库

#### 创建数据库

```sql
CREATE DATABASE ids_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'ids_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON ids_db.* TO 'ids_user'@'localhost';
FLUSH PRIVILEGES;
```

#### 导入数据库数据

```bash
# Windows
mysql -u root -p ids_db < "C:\Users\lenovo\Desktop\03-基于深度学习的入侵检测系统网络流量分析系统IDS\IDS\ids_2026-03-17_143148.sql"

# Linux/macOS
mysql -u root -p ids_db < "IDS/ids_2026-03-17_143148.sql"
```

### 5. 配置系统

编辑 `IDS-Web/backend/app/core/config.py`：

```python
class Settings:
    # 数据库配置
    DATABASE_URL = "mysql+pymysql://ids_user:your_password@localhost/ids_db"
    
    # 模型目录
    MODEL_DIR = "../data/saved_models"
    DATA_DIR = "../data"
    UPLOAD_DIR = "../uploads"
    
    # 日志目录
    LOG_DIR = "./logs"
    
    # 安全配置
    SECRET_KEY = "your-secret-key-change-this"
    ALGORITHM = "HS256"
    ACCESS_TOKEN_EXPIRE_MINUTES = 1440  # 24小时
```

### 6. 准备数据集

系统支持CICIDS2017数据集：

1. **CICIDS2017**
   - 数据集已位于 `IDS-Web/backend/data/cicids2017/` 目录
   - 文件名: `CICIDS2017.csv`

### 7. 创建必要目录

```bash
# 在backend目录执行
mkdir -p saved_models
mkdir -p uploads
mkdir -p eval_results
mkdir -p logs
```

## 配置说明

### 网卡配置

#### Windows系统
```bash
# 查看可用网卡
netsh interface show interface

# 启用混杂模式（需要管理员权限）
netsh interface set interface "Ethernet" admin=enable
```

#### Linux系统
```bash
# 查看可用网卡
ip link show

# 启用混杂模式
sudo ip link set eth0 promisc on
```

### 防火墙配置

#### Windows防火墙
```bash
# 查看现有规则
netsh advfirewall firewall show rule name=all

# 删除IDS规则
netsh advfirewall firewall delete rule name="IDS_Adaptive_Block_192.168.1.100"
```

#### Linux防火墙（iptables）
```bash
# 查看现有规则
sudo iptables -L INPUT -n

# 删除IDS规则
sudo iptables -D INPUT -s 192.168.1.100 -j DROP
```

## 启动服务

### 1. 启动Web后端

```bash
cd IDS-Web/backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

参数说明：
- `--host 0.0.0.0`：监听所有网络接口
- `--port 8000`：服务端口
- `--reload`：代码修改后自动重载（开发模式）

### 2. 访问系统

打开浏览器访问：`http://localhost:8000`

默认管理员账号：
- 用户名：`admin`
- 密码：`123456`

**首次登录后请立即修改密码！**

## 常见问题

### 1. 数据库连接失败

**问题**：`Can't connect to MySQL server`

**解决方案**：
- 检查MySQL服务是否启动
- 验证数据库用户名和密码
- 确认防火墙允许3306端口

### 2. 流量捕获失败

**问题**：`Permission denied` 或 `Operation not permitted`

**解决方案**：
- Windows：以管理员身份运行终端
- Linux：使用`sudo`运行程序
- 确认网卡支持混杂模式

### 3. 模型加载失败

**问题**：`Model file not found` 或 `Unsupported model type`

**解决方案**：
- 检查模型文件路径是否正确
- 确认模型类型与训练类型匹配
- 重新训练模型并保存到正确位置

### 4. 端口被占用

**问题**：`Address already in use`

**解决方案**：
```bash
# Windows
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# Linux
lsof -i :8000
kill -9 <进程ID>
```

### 5. Chart.js加载失败

**问题**：404错误加载Chart.js

**解决方案**：
- 确认已安装Chart.js：`cd chart && npm install`
- 检查静态文件挂载是否正确
- 清除浏览器缓存
